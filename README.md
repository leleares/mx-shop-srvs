# 运行user_srv前，请先检查main.go文件中的ip地址是否是本机ip地址，否则consul会失去健康检查。
# 每一个商品会对应很多图片，由于sql不支持数组类型，一般的做法是将这些图片单独抽离成一张表，使用goodsId跟图片对应起来，但是每次查询商品信息时，都需要做join操作，性能较低，比较好的做法是在model中将图片字段声明为一个独特的切片类型，实现gorm提供的Scanner和Valuer方法，就如同sql有了数组数据类型，但底层还是字符串和切片的相互转换。
